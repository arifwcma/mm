# WCMA Social Media Monitor — Feasibility & Plan

## Project Goal

Monitor WCMA's Facebook page for comments on posts. Analyze each comment for offensiveness/complaint severity (0–10 scale). Trigger email alert when score exceeds a configurable threshold (default: 6). Later extend to Instagram and other platforms.

## Feasibility Assessment — Facebook

**Status: FEASIBLE**

### What Facebook Graph API Provides

1. `GET /{page-post-id}/comments` — read all comments on any page post
2. Webhooks (`feed` field subscription) — real-time push notifications when new comments arrive, avoiding polling
3. Polling fallback — rate limit ~200 calls/user/hour, sufficient for a single page

### Required Permissions

| Permission | Purpose |
|---|---|
| `pages_read_engagement` | Read comments on page posts |
| `pages_read_user_content` | Read user-generated content on the page |
| `pages_manage_metadata` | Subscribe to webhooks |
| `pages_show_list` | List pages the app is installed on |

### Key Constraints

1. Meta App Review required for Live mode (24–72 hrs typical approval)
2. Webhooks require a publicly accessible HTTPS endpoint (valid TLS cert, no self-signed)
3. Page Access Token must be generated by someone with MODERATE task or higher on the page
4. Long-lived tokens still need periodic refresh handling

## Architecture (High-Level)

```
Facebook Webhook → HTTPS Endpoint → Comment Fetcher → Sentiment Scorer → Threshold Check → Email Alert
```

### Sentiment Scoring Approach

- LLM-based scoring (OpenAI API or similar) preferred for nuanced detection of offensiveness, complaints, sarcasm
- VADER as lightweight fallback if LLM cost is a concern
- Score: 0 (benign) to 10 (highly offensive/complaint)

### Email Alert

- Python `smtplib` or a service like SendGrid/Mailgun
- Triggered when any comment scores above the configured threshold

## Test Resources & Infrastructure Needed

### 1. Meta Business Account (Test)

- Create a test Meta Business Account (or use WCMA's existing one in dev mode)
- This is the parent container for everything below

### 2. Facebook Test Page

- Create a dedicated test Facebook Page under the business account
- Name it something like "WCMA Dev Test Page"
- Will be used to create test posts and comments without touching the real WCMA page

### 3. Meta Developer App

- Create at https://developers.facebook.com/
- App type: "Business"
- Add "Facebook Login" and "Webhooks" products
- While in Development mode, only app-role users can interact — no App Review needed yet

### 4. App Roles / Test Users

- Add developer/tester accounts as app roles
- Create Meta Test Users (available in App Dashboard → Roles → Test Users)
- These test users will post comments on the test page to simulate real activity

### 5. Page Access Token (Long-Lived)

- Generate via Graph API Explorer or programmatically
- Must be from a user with MODERATE+ role on the test page
- Exchange short-lived token for long-lived (~60 days)

### 6. Webhook Endpoint (Dev)

- Option A: `ngrok` tunnel to localhost during development
- Option B: Small cloud VM / serverless function (AWS Lambda, GCP Cloud Function)
- Must be HTTPS with valid TLS certificate

### 7. Sentiment Analysis API Key

- OpenAI API key (or alternative LLM provider) for scoring
- Or local model if preferring zero external dependency

### 8. Email Service

- SMTP credentials (Gmail app password, Outlook, or WCMA mail server)
- Or API key for SendGrid/Mailgun

### 9. Python Environment

- Python 3.10+
- Virtual environment with dependencies: `requests`, `flask`/`fastapi`, `openai`, email libs

## Sequence to Set Up

1. Meta Business Account (test) or confirm existing
2. Create test Facebook Page
3. Create Meta Developer App
4. Configure app permissions + add test users
5. Generate Page Access Token
6. Set up local dev environment + ngrok
7. Get sentiment API key
8. Configure email credentials
9. Build & test
